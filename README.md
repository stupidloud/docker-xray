# docker-xray

Minimal Docker-based Xray setup with:

- `xray` for VLESS over XTLS
- `caddy` for ACME certificate issuance and a simple fallback site
- `php-fpm` for serving PHP content behind Caddy

This repository is intended for personal deployment and keeps the stack small.

## Files

- `docker-compose.yml`: starts `php`, `caddy`, and `xray`
- `config.json`: Xray config template
- `Caddyfile`: Caddy config template
- `init.sh`: replaces `{host}` and `{uuid}` placeholders
- `index.html`: sample static page
- `resolv.conf`: resolver override mounted into the Caddy container

## How it works

1. Caddy listens on port `80` and obtains TLS certificates for your domain.
2. Xray listens on port `443` with VLESS + XTLS.
3. Xray reuses the certificate files generated by Caddy.
4. Non-Xray traffic falls back to Caddy.

The Xray certificate paths are read from the shared `caddy_data` volume:

- `/data/caddy/certificates/acme-v02.api.letsencrypt.org-directory/{host}/{host}.crt`
- `/data/caddy/certificates/acme-v02.api.letsencrypt.org-directory/{host}/{host}.key`

## Prerequisites

- Docker
- Docker Compose
- A public domain name pointed to this server
- Ports `80/tcp`, `443/tcp`, and `443/udp` open

## Quick start

Replace placeholders with your domain and UUID:

```bash
chmod +x init.sh
./init.sh example.com 11111111-1111-1111-1111-111111111111
```

Prepare required local files:

```bash
mkdir -p www
cp index.html www/index.html
touch hosts
```

Start the stack:

```bash
docker compose up -d
```

## Serving content

The current `docker-compose.yml` mounts site files from:

```text
./www -> /srv
```

So your web content should live under `www/`.

If you want to serve PHP files, place them in `www/` as well. The repo already starts a `php:8-fpm` container and Caddy mounts the same directory.

## Configuration notes

### Domain

`init.sh` replaces `{host}` in:

- `config.json`
- `Caddyfile`

### UUID

`init.sh` replaces `{uuid}` in:

- `config.json`

Generate a UUID with a tool such as:

```bash
cat /proc/sys/kernel/random/uuid
```

## Client settings

The default Xray inbound is:

- protocol: `vless`
- port: `443`
- flow: `xtls-rprx-direct`
- transport: `tcp`
- security: `xtls`

Use your configured domain and UUID in the client.

## Known caveats

- `docker-compose.yml` mounts `./hosts:/etc/hosts`, so you should create a local `hosts` file before startup.
- The sample `index.html` in the repo root is not served directly unless copied into `www/`.
- `init.sh` modifies files in place. Run it on a fresh copy if you want to keep the original placeholders.
